# 세모폰 (Semophone) — OKR (2026 Q1)

> 최종 업데이트: 2026-03-24 by [TCC]

## Objective 1: 운영 안정성 강화 및 데이터 동적 전환

v6.0 랜딩과 주요 기능이 운영 중이나, 매장 데이터가 정적 파일에 의존하고 있어 관리 효율이 낮음. Firestore 기반 동적 데이터로 전환하여 운영 효율을 높인다.

- KR1: 전국 40개 매장 데이터 Firestore 이관 완료 — 현재: data/stores.ts 정적 파일 / 목표: Firestore 컬렉션으로 100% 이관
- KR2: 매장 상세 페이지 (/stores/[id]) 구현 — 현재: 미구현 / 목표: 40개 매장 개별 상세 페이지 라이브
- KR3: 관리자 대시보드에서 매장 CRUD 가능 — 현재: 관리자 패널 존재하나 매장 편집 제한적 / 목표: 관리자가 매장 추가/수정/삭제 가능

## Objective 2: 사용자 참여 및 커뮤니티 기능 확장

매장 정보 제공을 넘어 사용자들이 직접 참여하는 리뷰/평가 시스템을 도입하여 재방문율을 높인다.

- KR1: 사용자 매장 리뷰 기능 구현 — 현재: 미구현 / 목표: 별점 + 텍스트 리뷰 시스템 라이브
- KR2: PWA 푸시 알림 기능 활성화 — 현재: PWA 기반 구축 완료, 푸시 미구현 / 목표: 매장 프로모션 알림 발송 가능

## Objective 3: 배포 인프라 현대화

Vercel에서 NCP Docker 기반으로의 배포 전환을 준비한다.

- KR1: NCP Docker 배포용 Dockerfile 작성 및 로컬 테스트 통과 — 현재: 미착수 / 목표: docker build + docker run 정상 동작
- KR2: CI/CD 파이프라인 구성 (GitHub Actions → NCP) — 현재: 미착수 / 목표: main 브랜치 push 시 자동 배포 파이프라인 구축
